# Step 2: Split & Extract SAP Actions

## Intent
Split multi-action corrective action entries into separate rows, then extract
structured data (resolved date, license number, destination station) from the
action text using regex patterns.

## Input
Cleaned SAP DataFrame from Step 1 with columns:
Tail, Flight Number, From Station, To Station, Date, Description, Defect, Corrective Action

## Processing Rules

### 1. Count Actions Per Row
Count occurrences of "DEFECT ACTION" in each row's Corrective Action text.
Store as `Action_Count` column.

### 2. Split Corrective Action
Split the Corrective Action column on the pattern `DEFECT ACTION` using a
lookahead regex: `(?=DEFECT ACTION)`. This splits WITHOUT removing the keyword.
Then explode the resulting list into individual rows (one row per action block).

### 3. Remove Empty Rows
After splitting, remove any rows where Corrective Action is empty or whitespace-only.

### 4. Extract Resolved Date
Extract the date from pattern: `ACTION ENTRY DATE & TIME:\s*(\d{2}\.\d{2}\.\d{4})`
- Parse as DD.MM.YYYY format
- Store in new column `Resolved date` formatted as DD-MM-YYYY string
- Rows without a matching date get NaN

### 5. Mark Rows for Highlighting
Create boolean column `Needs_Highlight`:
- True if the row contains "DEFECT ACTION" AND the original `Action_Count` > 1
- This identifies rows that were split from multi-action entries

### 6. Extract License Number
Extract from pattern: `LICENSE/CAAS NO:\s*([^(\n\r]+)`
- Store in new column `LIC NO`
- Strip trailing whitespace

### 7. Extract To Station
Extract 3-character code from last parentheses before line end:
`\((\w{3})\)\s*(?:\r?\n|$)`
- This overwrites the existing `To Station` column with the extracted value
- Rows without a match retain their original To Station value

## Output
Expanded DataFrame with additional columns: Action_Count, Resolved date,
Needs_Highlight, LIC NO. Row count increases due to action splitting.

## Review Criteria
- Row count should be >= input row count (splitting only adds rows)
- All Resolved date values should match DD-MM-YYYY format or be NaN
- LIC NO should not have leading/trailing whitespace
- Needs_Highlight should be True only for split rows (Action_Count > 1)
