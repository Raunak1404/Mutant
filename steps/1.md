# Step 1: Load & Clean SAP Data

## Intent
Load the SAP Raw Data Excel file for the SQ A350 fleet and perform initial cleaning:
select only the required columns, remove rows with missing defect/action data,
and filter out non-code descriptions (keeping only ATA/task codes).

## Input
- SAP Raw Data Excel file (e.g., `(SAP RAW DATA) SQ A350 FLEET 01-31 JAN 2026.xlsx`)
- Expected columns include: Tail, Flight Number, From Station, To Station, Date,
  Description, Defect Text1, ACTION Text1

## Processing Rules

### 1. Column Selection
Select exactly these 8 columns from the input:
- `Tail`
- `Flight Number`
- `From Station`
- `To Station`
- `Date`
- `Description`
- `Defect Text1`
- `ACTION Text1`

### 2. Drop Null Rows
Remove any row where `Defect Text1` OR `ACTION Text1` is null/NaN.

### 3. Filter Description Column
The Description column contains ATA codes (e.g., "Z200", "CDL001") mixed with
free-text descriptions (e.g., "WING FOUND DAMAGED", "DOOR INSPECTION").

**Keep only rows with valid codes:**
- Strip whitespace from Description
- Remove rows where Description contains standalone words of 4+ letters
  (regex: `\b[A-Za-z]{4,}\b`) — this removes free-text descriptions
- Then keep only rows where Description contains at least one digit (`\d`)

### 4. Rename Columns
- `Defect Text1` → `Defect`
- `ACTION Text1` → `Corrective Action`

## Output
Cleaned DataFrame with 8 columns (2 renamed), fewer rows after null/description filtering.

## Review Criteria
- No null values in Defect or Corrective Action columns
- All Description values should contain at least one digit
- No Description values should contain standalone words of 4+ letters
- Column count should be exactly 8
