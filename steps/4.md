# Step 4: Format SAP Output

## Intent
Rearrange columns into the final reporting order and add empty columns for
manual data entry by the vetting team. Apply yellow highlighting to rows
that were split from multi-action entries.

## Input
Classified SAP DataFrame from Step 3 with all columns from previous steps.

## Processing Rules

### 1. Rearrange Columns
Reorder existing columns to this exact sequence:
1. Tail
2. Flight Number
3. From Station
4. To Station
5. Date
6. Resolved date
7. Description
8. Defect
9. Corrective Action
10. LIC NO
11. Defect_Category
12. Needs_Highlight
13. Action_Count

### 2. Add Empty Manual-Entry Columns
Add the following columns (in order) with empty string values for all rows:
1. MHR
2. CHARGEABLE (YES/UNSURE)
3. REMARK
4. GPU
5. NOCO
6. BM
7. NITRO CART
8. CRANE
9. CH,PICKER
10. ENTERED BY
11. DATE UPDATED
12. PPW VET BY
13. REMARK.1
14. SHAIK'S REMARK
15. VETTER'S REMARK
16. RAY'S NOTE
17. INITIAL VET BY
18. DATE VETTED
19. CHECKED BY
20. DATE

### 3. Apply Styling Metadata
Preserve `Needs_Highlight` and `Action_Count` columns in the data but exclude
them from the final visible output. These are used by the export step (Step 6)
to apply yellow background highlighting to split rows.

## Output
Formatted DataFrame with 33 columns total (13 data + 20 empty manual columns).
Ready for export with styling metadata preserved.

## Review Criteria
- Total column count should be 33
- All 20 manual-entry columns should exist and be empty strings
- Column order should match the specification exactly
- Needs_Highlight and Action_Count columns must be preserved for styling
