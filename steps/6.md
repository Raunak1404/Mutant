# Step 6: Package & Export

## Intent
Combine outputs from the SAP track (Step 4) and ESJC track (Step 5) into
final styled Excel files. Apply visual formatting (yellow highlights for
split SAP rows) and exclude internal metadata columns from the output.

## Input
- SAP formatted DataFrame from Step 4 (passed as primary input_path)
- ESJC processed DataFrame from Step 5 (passed via libraries_dir as `esjc_output.xlsx`)

## Processing Rules

### 1. Load Both Inputs
- Read SAP data from input_path (Step 4 output)
- Read ESJC data from `esjc_output.xlsx` in libraries_dir

### 2. SAP Excel Styling
- Identify rows where `Needs_Highlight` is True
- Apply yellow background (`#FFFF00`) to ALL cells in those rows
- Remove `Action_Count` and `Needs_Highlight` columns from visible output
- Write to sheet named `SAP_Cleaned_Data`

### 3. ESJC Excel Output
- Write ESJC data as-is (no special styling needed)
- Write to sheet named `ESJC_Cleaned_Data`

### 4. Produce Two Output Files
- `output_sap.xlsx` — SAP data with yellow highlighting
- `output_esjc.xlsx` — ESJC data

The step writes both files. The primary output_path gets the SAP file.
The ESJC file is written alongside it (same directory, different name).

## Output
Two styled Excel files ready for the vetting team.

## Review Criteria
- SAP output should NOT contain Action_Count or Needs_Highlight columns
- Yellow highlighting should only appear on previously-split rows
- ESJC output should match Step 5 output exactly
- Both files should be valid .xlsx format
